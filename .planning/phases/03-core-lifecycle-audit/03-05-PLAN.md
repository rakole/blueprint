# Phase 3 - Plan 05

## Goal

Close the Phase 3 core lifecycle audit by checking `/blu-add-tests`, reconciling all Phase 3 bug reports, updating the bug index slice row, and confirming the no-fix boundary.

## Scope

- Compare `docs/commands/add-tests.md`, `commands/blu-add-tests.toml`, the validation skill, and add-tests runtime reference.
- Inspect add-tests report authoring context, report model validation, report write, verification render/write, and related tests.
- Verify add-tests is treated as evidence-backed repo-test generation in the product, while this milestone only documents defects and does not add or edit tests.
- Reconcile Phase 3 summaries and bug docs into `docs/bugs/INDEX.md`.
- Run final `git status --short` boundary check.

## Tasks

### Task 1: Audit add-tests contract alignment

#### Read First

- `docs/commands/add-tests.md`
- `commands/blu-add-tests.toml`
- `skills/blueprint-phase-validation/SKILL.md`
- `skills/blueprint-phase-validation/references/add-tests-runtime-contract.md`
- `docs/RUNTIME-REFERENCE.md`
- `.planning/phases/03-core-lifecycle-audit/03-CONTEXT.md`

#### Action

- Build a checklist with these exact expected add-tests requirements:
  - saved execution summaries are required before generating or updating tests
  - existing verification or UAT evidence is required before report authoring proceeds
  - broad or ambiguous test generation requires classification, scope, and test-plan gates
  - verification updates use `blueprint_phase_validation_render` and `blueprint_phase_validation_write`
  - durable report authoring uses `report.add-tests`, `blueprint_artifact_report_authoring_context`, `blueprint_artifact_report_validate_model`, and `blueprint_artifact_report_write`
  - Markdown report fallback is rejected for `report.add-tests`
  - final routing reports verification and report persistence outcomes from tool returns
- Compare command docs, manifest, skill, runtime reference, and runtime-reference row for material drift.
- File a bug if drift can cause test generation from chat memory, broad unsafe mutation, invalid report persistence, or misleading follow-up routing.

#### Acceptance Criteria

- `rg -n "saved execution summaries|report\\.add-tests|artifact_report_authoring_context|artifact_report_validate_model|artifact_report_write|phase_validation_render|Markdown report fallback|classification" docs/commands/add-tests.md commands/blu-add-tests.toml skills/blueprint-phase-validation/SKILL.md skills/blueprint-phase-validation/references/add-tests-runtime-contract.md docs/RUNTIME-REFERENCE.md` finds expected anchors.
