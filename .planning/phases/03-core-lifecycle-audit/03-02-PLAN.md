# Phase 3 - Plan 02

## Goal

Audit `/blu-plan-phase` as the bridge from saved discovery artifacts into executable plans, with special attention to config-aware readiness gates, structured model authoring, validation repair, and state truth after persistence.

## Scope

- Compare `docs/commands/plan-phase.md`, `commands/blu-plan-phase.toml`, the planning skill, and the runtime reference.
- Inspect the `phase.plan` contract, base schema, runtime-narrowed authoring context, validation, write, and plan-set validation handlers.
- Verify plan-check and no-subagent fallback rules are present and tested.
- Run targeted planning tests or document the environment blocker.
- Create bug reports only for confirmed or likely defects.

## Tasks

### Task 1: Audit plan-phase contract alignment

#### Read First

- `docs/commands/plan-phase.md`
- `commands/blu-plan-phase.toml`
- `skills/blueprint-phase-planning/SKILL.md`
- `skills/blueprint-phase-planning/references/plan-phase-runtime-contract.md`
- `docs/RUNTIME-REFERENCE.md`
- `docs/MCP-TOOLS.md`
- `.planning/phases/03-core-lifecycle-audit/03-CONTEXT.md`

#### Action

- Build a checklist with these exact expected plan-phase requirements:
  - `phase_research_status.planningReadiness` is the authoritative pre-draft gate.
  - Actual saved context, research, UI, validation, review, and existing plan content are read when present.
  - The command reads `phase.plan` and `blueprint_phase_plan_authoring_context` before drafting.
  - Plan models are validated with `blueprint_phase_plan_validate_model`.
  - Writes use `blueprint_phase_plan_write` with `validationMode: "strict"` and `authoringMode: "model-only"`; Markdown `content` fallback is not allowed.
  - Final saved plans are checked with `blueprint_phase_plan_validate`.
  - Checker review is bounded and config-gated.
  - State is synced after final persistence.
- Compare the command doc, manifest, skill, runtime reference, and runtime-reference row for material drift.
- File a bug if drift can lead to invalid plans, dropped decisions, stale routing, unsafe overwrite, or misleading downstream executors.

#### Acceptance Criteria

- `rg -n "planningReadiness|phase_plan_authoring_context|phase_plan_validate_model|phase_plan_write|authoringMode: \"model-only\"|phase_plan_validate|base: \"synced\"" docs/commands/plan-phase.md commands/blu-plan-phase.toml skills/blueprint-phase-planning/SKILL.md skills/blueprint-phase-planning/references/plan-phase-runtime-contract.md docs/RUNTIME-REFERENCE.md` finds the expected anchors.
- Any new bug report cites the exact mismatched contract surfaces in `## Evidence`.
