# Phase 3 - Plan 04

## Goal

Audit summary-backed validation and conversational UAT so Blueprint does not validate from chat memory, does not mark phases complete from incomplete evidence, and keeps UAT resumability and roadmap/state sync grounded in saved artifacts.

## Scope

- Compare `validate-phase` and `verify-work` command docs, manifests, validation skill text, and runtime references.
- Inspect `phase.verification` and `phase.uat` model contracts and validation handlers.
- Verify completed summaries are required for validation and ready verification is required for UAT.
- Verify UAT artifacts preserve resume state, checkpoint markers, structured gaps, blocked-prerequisite separation, and roadmap/state sync behavior.
- Run targeted validation/UAT tests or document the environment blocker.
- Create bug reports only for confirmed or likely defects.

## Tasks

### Task 1: Audit validate-phase contract alignment

#### Read First

- `docs/commands/validate-phase.md`
- `commands/blu-validate-phase.toml`
- `skills/blueprint-phase-validation/SKILL.md`
- `skills/blueprint-phase-validation/references/validate-phase-runtime-contract.md`
- `docs/PHASE-LIFECYCLE.md`
- `docs/RUNTIME-REFERENCE.md`
- `.planning/phases/03-core-lifecycle-audit/03-CONTEXT.md`

#### Action

- Build a checklist with these exact expected validate-phase requirements:
  - completed execution summaries are the validation baseline
  - State A/B/C behavior is explicit
  - existing verification artifacts are read before reuse or overwrite decisions
  - authoring uses `phase.verification`, validation authoring context, validation model validation, and model-only write
  - `PASS` is forbidden while unresolved gap, deferred-test, or repair signals remain
  - test-generation gaps route to `/blu-add-tests <phase>` and implementation/behavior gaps route to `/blu-audit-fix <phase>` when appropriate
  - state update uses synced artifact truth
- Compare command docs, manifest, skill, runtime reference, and runtime-reference row for material drift.
- File a bug if drift can cause validation from stale evidence, false PASS state, hidden blockers, or misleading next-safe-action routing.

#### Acceptance Criteria

- `rg -n "State A|State B|State C|phase\\.verification|phase_validation_authoring_context|phase_validation_validate_model|authoringMode: \"model-only\"|Gate State|/blu-add-tests|/blu-audit-fix" docs/commands/validate-phase.md commands/blu-validate-phase.toml skills/blueprint-phase-validation/SKILL.md skills/blueprint-phase-validation/references/validate-phase-runtime-contract.md docs/RUNTIME-REFERENCE.md` finds expected anchors.
- Any new bug report cites the exact mismatched contract surfaces in `## Evidence`.

### Task 2: Audit verify-work UAT contract alignment

#### Read First

- `docs/commands/verify-work.md`
- `commands/blu-verify-work.toml`
- `skills/blueprint-phase-validation/SKILL.md`
- `skills/blueprint-phase-validation/references/verify-work-runtime-contract.md`
- `src/mcp/artifact-contracts/schemas/phase.uat.model.schema.json`
- `tests/verify-work-roadmap-sync.test.ts`

#### Action

- Build a checklist with these exact expected verify-work requirements:
  - saved execution summaries plus ready verification are required before UAT persistence
  - existing valid incomplete UAT is resumed or reused by default
  - malformed UAT is not resumed blindly
  - the UAT queue is derived from saved summaries and verification evidence
  - plain user responses are classified into pass, skipped, blocked, or issue
  - blocked prerequisites are separate from code gaps
  - `XX-UAT.md` preserves `**Resume State:**` and `**Checkpoint:**`
  - roadmap completion changes only when saved summary, verification, and UAT evidence are all valid
- Compare command docs, manifest, skill, runtime reference, schemas, and tests for material drift.
- File a bug if drift can lose UAT evidence, mark a phase complete prematurely, or route around a resumable checkpoint.

#### Acceptance Criteria
