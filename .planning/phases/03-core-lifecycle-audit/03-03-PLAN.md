# Phase 3 - Plan 03

## Goal

Audit `/blu-execute-phase` so saved plans are selected deterministically, executed in safe wave order, summarized through the schema-first summary path, and routed onward without a premature phase-completion claim.

## Scope

- Compare `docs/commands/execute-phase.md`, `commands/blu-execute-phase.toml`, the execution skill, and runtime reference.
- Inspect execution target selection, lower-wave blockers, summary indexing, summary authoring context, model validation, summary write, and state sync handlers.
- Verify `PARTIAL` and `BLOCKED` summaries remain pending carry-forward evidence rather than completed plans.
- Run targeted execution summary tests or document the environment blocker.
- Create bug reports only for confirmed or likely defects.

## Tasks

### Task 1: Audit execute-phase contract alignment

#### Read First

- `docs/commands/execute-phase.md`
- `commands/blu-execute-phase.toml`
- `skills/blueprint-phase-execution/SKILL.md`
- `skills/blueprint-phase-execution/references/execute-phase-runtime-contract.md`
- `skills/blueprint-phase-execution/references/long-running-execution-profile.md`
- `docs/PHASE-LIFECYCLE.md`
- `docs/RUNTIME-REFERENCE.md`
- `.planning/phases/03-core-lifecycle-audit/03-CONTEXT.md`

#### Action

- Build a checklist with these exact expected execute-phase requirements:
  - `blueprint_phase_execution_targets` selects default, `--wave`, and `--gaps-only` targets.
  - `gapClosurePlans` is the source of truth for `--gaps-only`.
  - lower-wave pending plans block later-wave execution.
  - selected plans are read through `blueprint_phase_plan_read`.
  - existing summaries are read before reuse or overwrite decisions.
  - summary authoring uses `phase.summary`, summary authoring context, summary model validation, and model-only write.
  - `COMPLETED` is the only summary status that closes a plan.
  - valid `PARTIAL` and `BLOCKED` summaries remain pending.
  - execute-phase does not persist reports and does not claim phase completion before validation/UAT.
- Compare command docs, TOML prompts, skill text, runtime references, and runtime-reference rows.
- File a bug if drift can cause unsafe execution order, stale summary reuse, lost carry-forward evidence, invalid summaries, or misleading next actions.

#### Acceptance Criteria

- `rg -n "execution_targets|gapClosurePlans|lowerWavePendingPlans|phase_summary_authoring_context|phase_summary_validate_model|phase_summary_write|PARTIAL|BLOCKED|does not persist reports" docs/commands/execute-phase.md commands/blu-execute-phase.toml skills/blueprint-phase-execution/SKILL.md skills/blueprint-phase-execution/references/execute-phase-runtime-contract.md docs/RUNTIME-REFERENCE.md` finds the expected anchors.
- Any new bug report cites the exact mismatched contract surfaces in `## Evidence`.

### Task 2: Audit summary MCP handlers and execution tests

#### Read First

- `src/mcp/tools/phase.ts`
- `src/mcp/artifact-contracts/index.ts`
- `src/mcp/artifact-contracts/schemas/phase.summary.model.schema.json`
- `tests/execute-phase-summary-tools.test.ts`
- `tests/execute-phase-metadata.test.ts`

#### Action

- Inspect summary and target handlers for:
  - completed versus pending summary classification
  - dependency and lower-wave blockers
  - overwrite candidate and overlap warnings
  - summary authoring context prerequisite blockers
  - model-only write rejection of Markdown `content`
  - validation of status/readiness/completion truth table
  - synced state update expectations in tests/docs
- Run targeted tests:
  - `npx tsx --test tests/execute-phase-summary-tools.test.ts tests/execute-phase-metadata.test.ts`
- If tests fail, classify whether the failure is defect evidence, an environment blocker, or unrelated pre-existing breakage.

#### Acceptance Criteria

- `rg -n "blueprintPhaseExecutionTargets|blueprintPhaseSummaryAuthoringContext|blueprintPhaseSummaryValidateModel|blueprintPhaseSummaryWrite|PARTIAL|BLOCKED|pendingPlans|content is invalid" src/mcp/tools/phase.ts tests/execute-phase-summary-tools.test.ts tests/execute-phase-metadata.test.ts` finds handler and test evidence.
- The summary records targeted test pass/fail output or a concrete environment blocker.
