# Phase 9 Plan 01: Inventory And Status Freshness Audit

## Goal

Create the Phase 9 inventory baseline by reconciling every real bug report against `docs/bugs/INDEX.md`, separating active defects from repaired or historical items, and handling BPBUG-004 status freshness before any repair priority ranking happens.

## Scope

In scope:
- BPBUG-001 through BPBUG-005 metadata and report-readiness review.
- `BPBUG-000` exclusion from real totals.
- BPBUG-004 current-status reconciliation using the quick repair and Phase 7 validation evidence.
- `docs/bugs/INDEX.md` additions needed to distinguish active candidates from repaired/history items.
- `09-INVENTORY-REVIEW.md` and `09-01-SUMMARY.md`.

Out of scope:
- Changing source, manifests, skills, tests, generated assets, runtime behavior, `.blueprint/`, installed extensions, host-global state, branches, remotes, PRs, or git history.
- Marking duplicates; Plan 02 owns duplicate policy.
- Assigning repair priority bands; Plan 03 owns priority.
