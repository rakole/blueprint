# Phase 2 - Plan 03

## Goal

Audit bootstrap, map-first, settings/profile, and health readiness behavior so Blueprint initializes and repairs `.blueprint/` state through the documented MCP/config substrate without unsafe prompt-only or manual persistence.

## Scope

- Compare `new-project` and `map-codebase` docs/manifests/skills against `blueprintProjectStatus()` and bootstrap gate behavior.
- Compare settings/profile/health docs/manifests/skills against config normalization and state repair tools.
- Inspect targeted bootstrap/config tests.
- Create bug reports only for confirmed or likely defects.

## Tasks

### Task 1: Audit map-first bootstrap gates for `new-project` and `map-codebase`

#### Read First

- `docs/commands/new-project.md`
- `docs/commands/map-codebase.md`
- `commands/blu-new-project.toml`
- `commands/blu-map-codebase.toml`
- `skills/blueprint-bootstrap/SKILL.md`
- `skills/blueprint-map/SKILL.md`
- `src/mcp/tools/project.ts`
- `tests/new-project.test.ts`
- `tests/map-codebase.test.ts`

#### Action

- Verify `new-project` docs/manifests/skills require:
  - reading project status before first write;
  - unmapped brownfield and `mapping-incomplete` routing to `/blu-map-codebase`;
  - `mapped-only` preserving `.blueprint/codebase/*.md` and allowing project bootstrap;
  - sufficient `bootstrapSeed` before persistence;
  - no manual `.blueprint/` hand-editing around MCP ownership.
- Verify `map-codebase` docs/manifests/skills require:
  - allowing brownfield codebase-only first write;
  - writing only the seven `.blueprint/codebase/*.md` artifacts;
  - reuse-before-refresh posture and explicit confirmation for replacement;
  - route to `/blu-new-project` after successful mapped-only state.
- File bugs for material contradictions between docs/manifests/skills/source/tests.

#### Acceptance Criteria

- `rg -n "mapping-incomplete|mapped-only|brownfield|bootstrapSeed|blueprint_project_init|blueprint_codebase_artifact_write|/blu-map-codebase|/blu-new-project" docs/commands/new-project.md docs/commands/map-codebase.md commands/blu-new-project.toml commands/blu-map-codebase.toml src/mcp/tools/project.ts tests/new-project.test.ts tests/map-codebase.test.ts` finds evidence.
- Any bug report cites at least one expected contract file and one actual source/test/manifest evidence file.

### Task 2: Audit settings, set-profile, and health config governance

#### Read First

- `docs/commands/settings.md`
- `docs/commands/set-profile.md`
- `docs/commands/health.md`
- `commands/blu-settings.toml`
- `commands/blu-set-profile.toml`

- `src/mcp/tools/config.ts`
- `src/mcp/tools/state.ts`
- `tests/help-progress-health.test.ts`

#### Action

- Verify config contracts use `.blueprint/config.json` as project runtime config and optional `~/.<host>/blueprint/defaults.json` as user defaults.
- Verify `blueprint_config_get`, `blueprint_config_set`, and `blueprint_config_set_profile` behavior aligns with docs/manifests for scopes, normalized v2 config, allowed model profiles, warnings, and project-local repair.
- Verify `health` repair language requires confirmation before config/state writes and routes partial repos to `/blu-health`.
- File bugs for material config default, scope, repair, or profile mismatches.

#### Acceptance Criteria

- `rg -n "blueprint_config_get|blueprint_config_set|blueprint_config_set_profile|scope|defaults|model_profile|partial|/blu-health" docs/commands/settings.md docs/commands/set-profile.md docs/commands/health.md commands/blu-settings.toml commands/blu-set-profile.toml commands/blu-health.toml src/mcp/tools/config.ts src/mcp/tools/state.ts tests/help-progress-health.test.ts` finds evidence.
