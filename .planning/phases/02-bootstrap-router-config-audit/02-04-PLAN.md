# Phase 2 - Plan 04

## Goal

Close the Phase 2 discovery slice by reconciling all bug reports and the bug index, validating the no-fix boundary, and preserving a clean handoff to validation/UAT.

## Scope

- Read Plan 01 through Plan 03 summaries and every Phase 2 bug report.
- Update `docs/bugs/INDEX.md` Phase 2 slice coverage with examined surfaces, deferred surfaces, and bug ids.
- Verify every listed Phase 2 bug file exists and follows the template.
- Verify no source, manifest, skill, test, build, generated asset, `.blueprint/`, or host-global state fixes were applied.

## Tasks

### Task 1: Reconcile bug index rows against bug report files

#### Read First

- `docs/bugs/INDEX.md`
- `docs/bugs/TEMPLATE.md`
- `.planning/phases/02-bootstrap-router-config-audit/02-01-SUMMARY.md`
- `.planning/phases/02-bootstrap-router-config-audit/02-02-SUMMARY.md`
- `.planning/phases/02-bootstrap-router-config-audit/02-03-SUMMARY.md`
- all `docs/bugs/BPBUG-*.md` files

#### Action

- List every real Phase 2 bug report with `discovery_phase: 2`.
- Ensure each has an index row with id, title, severity, confidence, surface, status, discovery phase, impact, likely cause, and report link.
- Ensure index rows do not count `BPBUG-000` as real.
- If a Phase 2 summary records no defect for a surface, include that surface in the Phase 2 slice coverage row with no bug id.

#### Acceptance Criteria

- `rg -n "discovery_phase: 2" docs/bugs/BPBUG-*.md` finds every real Phase 2 bug report when any exist.
- `rg -n "Phase 2|Bootstrap Router Config Audit" docs/bugs/INDEX.md` finds the final slice row.

### Task 2: Verify template and evidence completeness

#### Read First

- `docs/bugs/TEMPLATE.md`
- every real Phase 2 `docs/bugs/BPBUG-###-*.md`

#### Action

- For each real Phase 2 bug report, verify these required sections exist:
  - `## Summary`
  - `## Classification`
  - `## Expected Behavior`
  - `## Actual Behavior`
  - `## Impact`
  - `## Affected Files`
  - `## Evidence`
  - `## Verification Steps`
  - `## Likely Cause`
  - `## Suggested Fix Direction`
  - `## Uncertainty`
  - `## Related Bugs`
  - `## No Fix Applied`
- Verify `## Evidence` cites exact files, commands, tests, line references where useful, or contract mismatches.
- Verify `## Uncertainty` is explicit and does not hide weak evidence.
- Verify no report includes source-fix language such as "fixed", "patched", or "updated source" as a completed action unless explicitly saying no fix was applied.

#### Acceptance Criteria

- `rg -n "^## Summary|^## Classification|^## Expected Behavior|^## Actual Behavior|^## Impact|^## Affected Files|^## Evidence|^## Verification Steps|^## Likely Cause|^## Suggested Fix Direction|^## Uncertainty|^## Related Bugs|^## No Fix Applied" docs/bugs/BPBUG-*.md` finds required headings in Phase 2 reports when any exist.
- `rg -n "No source, manifest, skill, test, generated asset, or runtime behavior fix was applied during this discovery milestone." docs/bugs/BPBUG-*.md` finds the no-fix sentence in Phase 2 reports when any exist.
