# Phase 2 - Plan 02

## Goal

Audit the implemented-command catalog and runtime substrate so Blueprint does not expose commands whose manifest, primary skill, command spec, required MCP tools, or runtime-contract resource is missing or contradictory.

## Scope

- Compare `docs/COMMAND-CATALOG.md` declared statuses with live catalog derivation in `src/mcp/tools/project.ts`.
- Verify required MCP tool parsing and registration against `docs/MCP-TOOLS.md` and `src/mcp/server.ts`.
- Verify primary skill resolution against `skills/*/SKILL.md` and `src/mcp/runtime-vocabulary.ts`.
- Verify runtime-contract resource exposure in `src/mcp/command-resources.ts`.
- Run or inspect `tests/command-catalog.test.ts` and `tests/command-contract-docs.test.ts`.

## Tasks

### Task 1: Audit declared implemented status against runtime catalog derivation

#### Read First

- `docs/COMMAND-CATALOG.md`
- `src/mcp/tools/project.ts`
- `src/mcp/server.ts`
- `src/mcp/runtime-vocabulary.ts`
- `tests/command-catalog.test.ts`

#### Action

- Inspect `buildCommandCatalogEntry()` and record the exact runtime requirements for `implemented: true`:
  - command spec exists at `docs/commands/<command>.md`;
  - manifest exists at `commands/blu-<command>.toml`;
  - primary skill resolves under `skills/<skill>/SKILL.md`;
  - every required MCP tool parsed from the spec exists in the registered tool set;
  - `status === "implemented"`.
- Compare this with `docs/COMMAND-CATALOG.md` declared `implemented` rows for Phase 2-relevant commands: `new-project`, `settings`, `set-profile`, `help`, `progress`, `health`, `map-codebase`, and `next`.
- If any declared implemented command is runtime `repairing`, `blocked`, or missing required substrate, create a bug classified as a command catalog/status contract defect.

#### Acceptance Criteria

- `rg -n "buildCommandCatalogEntry|manifestExists|skillExists|requiredToolsSatisfied|blockedBy|implemented: status === \"implemented\"" src/mcp/tools/project.ts` finds catalog derivation evidence.
- `rg -n "\`new-project\`|\`settings\`|\`set-profile\`|\`help\`|\`progress\`|\`health\`|\`map-codebase\`|\`next\`" docs/COMMAND-CATALOG.md` finds the Phase 2 declared rows.

### Task 2: Audit runtime-contract resource projection

#### Read First

- `src/mcp/command-resources.ts`
- `docs/RUNTIME-REFERENCE.md`
- `docs/commands/help.md`
- `docs/commands/progress.md`
- `docs/commands/next.md`
- `tests/command-catalog.test.ts`

#### Action

- Inspect `isExposedRuntimeContractCatalogEntry()` and `listBlueprintCommandRuntimeContractCommands()`.
- Confirm runtime-contract resources are exposed only when catalog entries are implemented and specs/runtime-reference rows exist.
- Confirm any intentional runtime-contract exclusion, such as `review`, is documented and tested consistently.
- File a bug if an implemented Phase 2 command lacks a runtime contract unexpectedly, if a planned command is exposed, or if docs/tests disagree with runtime projection.

#### Acceptance Criteria

- `rg -n "isExposedRuntimeContractCatalogEntry|listBlueprintCommandRuntimeContractCommands|BLUEPRINT_COMMAND_RUNTIME_CONTRACT_EXCLUSIONS|implemented" src/mcp/command-resources.ts tests/command-catalog.test.ts` finds projection evidence.
- Any runtime-contract bug cites the command name, expected exposure, actual exposure, and the docs/source/test mismatch.

### Task 3: Run catalog and docs-contract probes

#### Read First

- `tests/command-catalog.test.ts`
- `tests/command-contract-docs.test.ts`
- `package.json`

#### Action

- If dependencies are installed, run:
  - `npx tsx --test tests/command-catalog.test.ts tests/command-contract-docs.test.ts`
- If dependencies are missing in a fresh worktree, run `npm ci` first before tests.
- Treat failing tests as evidence only after confirming the failure relates to catalog/substrate contracts and is not an environment issue.
- Create bug reports for confirmed or likely defects and update `docs/bugs/INDEX.md`.
- If no defect is found, update Phase 2 slice coverage with `catalog/substrate surfaces examined; none found in Plan 02`.

#### Acceptance Criteria

- The plan summary records whether the targeted test command passed, failed with defect evidence, or was blocked by environment.
- `rg -n "Phase 2|catalog|Plan 02" docs/bugs/INDEX.md` finds the slice coverage update.
