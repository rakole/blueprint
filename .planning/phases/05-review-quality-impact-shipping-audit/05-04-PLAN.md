# Phase 5 Plan 04: High-Risk Git Workflow Safety

## Goal

Audit `pr-branch`, `ship`, and `undo` for pre-mutation report/preview gates, dirty-tree and health blockers, confirmation posture, report persistence, source branch/history safety, remote fallback behavior, and implemented-only routing.

## Scope

In scope:
- `/blu-pr-branch` review-branch filtering, commit classification, preview, source branch preservation, and `pr-branch-latest`.
- `/blu-ship` local prep, optional push, optional PR creation, saved-evidence gate, report persistence, manual fallback, and state routing.
- `/blu-undo` safe revert-style behavior, dirty-tree/detached/merge blockers, dependency impact preview, report-before-mutate posture, and `undo-latest`.
- Maintenance skill, `pr-branch` runtime contract, report contracts, and metadata tests.

Out of scope:
- Creating branches, pushing, opening PRs, reverting commits, or mutating git history in the main worktree.
- Workspace/cleanup/update/reapply-patches maintenance surfaces, which belong to Phase 6.

## Tasks

### Task 1: Audit high-risk command contract alignment

#### Read First

- `docs/commands/pr-branch.md`
- `docs/commands/ship.md`
- `docs/commands/undo.md`
- `commands/blu-pr-branch.toml`
- `commands/blu-ship.toml`
- `commands/blu-undo.toml`
- `skills/blueprint-maintenance/SKILL.md`
- `skills/blueprint-maintenance/references/pr-branch-runtime-contract.md`
- `docs/MCP-TOOLS.md`
- `docs/RUNTIME-REFERENCE.md`
- `docs/ARTIFACT-SCHEMA.md`

#### Action

Verify these concrete target states:

- `/blu-pr-branch` reads project status/config/digest/report contract, classifies commits ahead of base as `code-only`, `blueprint-only`, `mixed`, or `empty-after-filter`, previews included/excluded paths and git mutations, requires confirmation, creates/replays only after confirmation, validates the review branch, and persists `pr-branch-latest`.
- `/blu-ship` hard-stops on dirty worktrees or missing base branch, uses saved evidence as a gate, previews exact push/PR commands, separates local prep from optional push and PR creation, handles missing `gh` with manual fallback, persists `ship-latest`, and updates state only after report write.
- `/blu-undo` hard-stops on dirty tree, detached HEAD, merge in progress, or missing target; previews candidate commits and stale evidence; persists `undo-latest` before approved `git revert` steps; and never uses hard reset or branch deletion.

#### Acceptance Criteria

- `rg -n "pr-branch-latest|ship-latest|undo-latest|confirmation|dirty working tree|git revert|manual fallback|source branch|report-overwrite|report-before|push|PR" docs/commands/pr-branch.md docs/commands/ship.md docs/commands/undo.md commands/blu-pr-branch.toml commands/blu-ship.toml commands/blu-undo.toml skills/blueprint-maintenance/SKILL.md skills/blueprint-maintenance/references/pr-branch-runtime-contract.md docs/MCP-TOOLS.md docs/RUNTIME-REFERENCE.md docs/ARTIFACT-SCHEMA.md` finds aligned contract anchors.
- Any missing report/preview, dirty-tree, confirmation, fallback, or source-branch/history safety behavior is either ruled out with citations or recorded as a bug candidate.

- `report.undo` to include selected scope, candidate commits, dependency-impact notes, digest inputs, branch state, and pending/approved revert commands.

Run or document:

```bash

```

#### Acceptance Criteria

- The targeted high-risk git metadata command exits 0, or the exact failure/environment blocker is recorded.
- Any missing report field or unsafe metadata alignment is documented as a bug candidate with exact file/test evidence.

## Threat Model

| Threat | Risk | Mitigation |
|--------|------|------------|
| T-01 | The audit mutates branches, remotes, PRs, or history. | Use static review and metadata tests in the main worktree; disposable local repos only if a material ambiguity remains. |
| T-02 | A high-risk command mutates before preview/report evidence is reviewable. | Audit report-before-mutate posture, confirmation gates, and report contract fields. |
| T-03 | Manual fallback or dirty-tree blockers are missing. | Check docs, manifests, skill/runtime references, and metadata tests for explicit blockers and fallback paths. |

## Must Haves

- Git and remote mutations are previewed and confirmation-gated.
- `undo` uses safe revert-style behavior and preserves history.
- Reports are durable before or alongside approved high-risk actions as the command contract requires.
- No branch, PR, remote, or git-history mutation is performed by this discovery audit.
