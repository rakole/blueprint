# Phase 5 Plan 03: Impact Analysis Depth And Scaling

## Goal

Audit `/blu-impact` for config provenance, scope resolution, context loading, file-surface classification, ownership/dependency coverage, generated/source mixed-scope warnings, report validation/write, output rendering, and large-repo graceful degradation.

## Scope

In scope:
- `blueprint_impact_config_get`, `blueprint_impact_scope_resolve`, `blueprint_impact_context_load`, `blueprint_impact_analyze`, `blueprint_impact_report_write`, and `blueprint_impact_output_render`.
- `docs/commands/impact.md`, `commands/blu-impact.toml`, `skills/blueprint-impact/SKILL.md`, and `impact-runtime-contract.md`.
- `report.impact` schema, impact fixtures, impact metadata tests, and impact tool tests.

Out of scope:
- Writing real `.blueprint/impact/` bundles in this main audit worktree unless the execution command intentionally uses a disposable temp root or `--no-write` style path.
- High-risk git shipping behavior, which is Plan 04.

## Tasks

### Task 1: Audit impact command and skill contract alignment

#### Read First

- `docs/commands/impact.md`
- `commands/blu-impact.toml`
- `skills/blueprint-impact/SKILL.md`
- `skills/blueprint-impact/references/impact-runtime-contract.md`
- `docs/MCP-TOOLS.md`
- `docs/RUNTIME-REFERENCE.md`
- `docs/ARTIFACT-SCHEMA.md`

#### Action

Verify these concrete target states:

- `/blu-impact` calls impact tools in order: config, scope, context, analyze, optional report write, output render.
- Config provenance includes built-in defaults, host-global defaults, project config, invocation config, overrides, unknown key warnings, errors, and config hash.
- Scope resolution supports staged, working-tree, range, base/head, explicit files, diff-file, CI, and description-only scopes with patch hash/fingerprint/confidence.
- Analysis keeps impact status, risk, confidence, surfaces, ownership, dependency graph, findings, obligations, unknowns, evidence, and normalized report separate.
- Report writes are bounded to `.blueprint/impact/<impact-id>/` and reject mismatched expected files, evidence ids, finding ids, or nested changed-file-bearing content outside expected files.
- Output rendering supports human, json, markdown, pr-comment, and summary modes without writing files.

#### Acceptance Criteria

- `rg -n "config_get|scope_resolve|context_load|impact_analyze|impact_report_write|output_render|risk|confidence|unknowns|ownership|dependency" docs/commands/impact.md commands/blu-impact.toml skills/blueprint-impact/SKILL.md skills/blueprint-impact/references/impact-runtime-contract.md docs/MCP-TOOLS.md docs/RUNTIME-REFERENCE.md docs/ARTIFACT-SCHEMA.md` finds aligned contract anchors.
- Any missing contract alignment or planned-only command recommendation is either ruled out with citations or recorded as a bug candidate.

### Task 2: Audit impact runtime behavior and scaling safeguards

#### Read First

- `src/mcp/tools/impact.ts`
- `src/mcp/artifact-contracts/index.ts`
- `src/mcp/artifact-contracts/schemas/report.impact.model.schema.json`
- `tests/impact-metadata.test.ts`
- `tests/impact-tools.test.ts`
- `tests/impact-fixtures.test.ts`
- `tests/fixtures/impact/`

#### Action

Inspect whether the impact runtime:

- Distinguishes missing ownership metadata, dependency unknowns, and low confidence from proof of safety.
- Flags generated/source mixed-scope conditions, command-substrate gaps, planned-command exposure gates, and path-level test/review/release/security/deployment/build obligations.
- Bounds expensive import/dependency scanning and handles large lockfiles or large diffs without misleading PASS confidence.
- Validates and canonicalizes report output before writing, with overwrite/reuse behavior visible.

Run or document:

```bash
npx tsx --test tests/impact-metadata.test.ts tests/impact-tools.test.ts tests/impact-fixtures.test.ts
```

#### Acceptance Criteria

- The targeted impact command exits 0, or the exact failure/environment blocker is recorded.
- Any impact PASS/WARN/BLOCK, risk, confidence, owner/dependency, or report-writing behavior that can mislead users is documented as a bug candidate with concrete code/test evidence.

### Task 3: Record impact findings

#### Read First

- `docs/bugs/TEMPLATE.md`
- `docs/bugs/INDEX.md`
- `.planning/phases/05-review-quality-impact-shipping-audit/05-PATTERNS.md`

#### Action

If a confirmed or likely impact defect is found:

- Create one `docs/bugs/BPBUG-###-short-slug.md` file.
- Set `discovery_phase: 5`.
- Classify the affected surface as command, skill, MCP tool, docs, tests, state artifacts, or cross-cutting.
- Include exact file paths, line-level references where useful, and command/test evidence.
- Include the exact no-fix sentence from the template.
- Add or update the matching row in `docs/bugs/INDEX.md`.

If no confirmed or likely defect is found, do not create a bug file. Record that outcome in `05-03-SUMMARY.md` during execution.

#### Acceptance Criteria

- If a bug is created, `rg -n "discovery_phase: 5|No source, manifest, skill, test, generated asset, or runtime behavior fix was applied" docs/bugs/BPBUG-*.md` finds it.
- `git status --short` shows Plan 03 changes limited to `docs/bugs/` and `.planning/phases/05-review-quality-impact-shipping-audit/`.

## Threat Model

| Threat | Risk | Mitigation |
|--------|------|------------|
| T-01 | Impact reports show high confidence while metadata is missing. | Audit owner/dependency/unknown/confidence behavior and fixture tests. |
| T-02 | Impact execution writes real report bundles in the audit worktree unnecessarily. | Prefer static review and existing tests; use disposable temp roots or no-write paths for probes. |
| T-03 | Large repo or large diff handling hides incomplete analysis. | Inspect bounded scanning and warnings for large lockfiles, import scans, and partial dependency graphs. |

## Must Haves

- Impact risk and confidence remain separate.
- Missing owners/dependencies are visible as warnings, unknowns, or blockers, never proof of safety.
- Report writes are validated, bounded, and overwrite-protected.
- Any bug is evidence-backed and no runtime fix is applied.
