# Phase 5 Plan 02: Remediation And Docs Mutation Safety

## Goal

Audit `code-review-fix`, `audit-fix`, and `docs-update` for bounded mutation safety, durable evidence, report validation, overwrite/confirmation gates, dry-run behavior, verification, and implemented-only state routing.

## Scope

In scope:
- Saved finding selection and `review.review-fix` persistence for `/blu-code-review-fix`.
- Audit-driven remediation, dry-run mode, report authoring context, and `report.audit-fix` model validation for `/blu-audit-fix`.
- Scoped repo-doc refresh, evidence-backed docs report behavior, and broad-refresh routing for `/blu-docs-update`.
- `src/mcp/tools/review.ts`, `src/mcp/tools/artifacts.ts`, `src/mcp/tools/state.ts`, and focused metadata/slice/regression tests.

Out of scope:
- Applying real source or docs fixes during this discovery phase.
- Creating commits, branches, or PRs.
- Impact-specific analysis, which is Plan 03.

## Tasks

### Task 1: Audit remediation command contract alignment

#### Read First

- `docs/commands/code-review-fix.md`
- `docs/commands/audit-fix.md`
- `docs/commands/docs-update.md`
- `commands/blu-code-review-fix.toml`

- `skills/blueprint-docs/SKILL.md`
- `docs/MCP-TOOLS.md`
- `docs/RUNTIME-REFERENCE.md`

#### Action

Verify these concrete target states:

- `/blu-code-review-fix` loads saved code-review findings through `blueprint_review_load_findings`, narrows selected target ids through `blueprint_review_authoring_context`, uses the same `targetIds` for validation and persistence, and never invents findings from git drift or chat memory.
- `/blu-audit-fix` classifies from saved evidence selected by `--source`, honors `--dry-run` as no repo mutation, passes `auditFixContext {source, severity, maxAttempts, dryRun, scopeFiles}` through authoring/validation/write, and rejects Markdown fallback for `report.audit-fix`.
- `/blu-docs-update` keeps selected docs, source files, tests, and digest-backed Blueprint artifacts explicit as truth sources; routes evidence-light broad refreshes to `/blu-map-codebase`; and persists `docs-update-latest` through the report substrate.

#### Acceptance Criteria

- `rg -n "targetIds|auditFixContext|dry-run|docs-update-latest|blueprint_artifact_report_write|blueprint_state_update|confirmation|next safe" docs/commands/code-review-fix.md docs/commands/audit-fix.md docs/commands/docs-update.md commands/blu-code-review-fix.toml commands/blu-audit-fix.toml commands/blu-docs-update.toml skills/blueprint-review/SKILL.md skills/blueprint-review/references/*.md skills/blueprint-docs/SKILL.md docs/MCP-TOOLS.md docs/RUNTIME-REFERENCE.md` finds expected anchors.
- Any missing confirmation, dry-run, overwrite, or report-before/after evidence contract is either ruled out with citations or recorded as a bug candidate.

- `report.audit-fix` rejects unsupported sources/severity, invalid max attempts, stale evidence, dry-run mutation claims, changed-file claims during dry-run, and unsupported Markdown `content`.
- `docs-update` evidence, report path, and state routing remain explicit enough to avoid broad unexpected docs/source writes.

#### Acceptance Criteria

## Threat Model

| Threat | Risk | Mitigation |
|--------|------|------------|
| T-01 | The audit applies real remediation or docs changes. | Use static reads, existing tests, dry-run reasoning, or disposable temp roots only; record cleanup if used. |
| T-02 | A flow mutates before durable evidence is reviewable. | Check report authoring, validation, overwrite, confirmation, and state-routing order. |
| T-03 | Dry-run reports can claim mutations. | Inspect `report.audit-fix` validation and tests for dry-run mutation rejection. |

## Must Haves

- Saved evidence is authoritative for remediation selection.
- Dry-run and confirmation semantics are explicit and test-backed.
- Report/state writes remain MCP-owned and evidence-backed.
- No real source/docs/runtime fixes are applied during audit execution.
