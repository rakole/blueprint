# Phase 7 Plan 05: Bug Index Reconciliation And No-Fix Closeout

## Goal

Reconcile all Phase 7 findings, update the bug index slice row, verify the discovery-only boundary, and prepare Phase 7 for validation without applying packaging, build, hook, install, generated asset, or runtime fixes.

## Scope

In scope:
- Phase 7 summaries from Plans 01 through 04.
- All Phase 7 `docs/bugs/BPBUG-###-*.md` reports created during execution.
- `docs/bugs/INDEX.md` bug rows, duplicate/related links, and Phase 7 slice coverage row.
- No-fix and git-status boundary verification.

Out of scope:
- Creating new source, manifest, test, hook, build, generated asset, install, or runtime fixes.
- Re-running broad integration tests unless needed to resolve unclear Phase 7 evidence.
- Final cross-phase priority review; that belongs to Phase 9.

## Tasks

### Task 1: Reconcile Phase 7 bug reports

#### Read First

- `docs/bugs/TEMPLATE.md`
- `docs/bugs/INDEX.md`
- `.planning/phases/07-host-packaging-build-hooks-audit/07-01-SUMMARY.md`
- `.planning/phases/07-host-packaging-build-hooks-audit/07-02-SUMMARY.md`

- `.planning/phases/07-host-packaging-build-hooks-audit/07-04-SUMMARY.md`

#### Action

For every Phase 7 bug report created by Plans 01 through 04:

- Verify frontmatter includes `discovery_phase: 7`, severity, confidence, affected surface, status, and report id.
- Verify body includes expected behavior, actual behavior, impact, evidence, reproduction or verification steps, likely cause, suggested fix direction, affected files, related bugs, uncertainty, and `No Fix Applied`.
- Verify each report states: `No source, manifest, skill, test, generated asset, or runtime behavior fix was applied during this discovery milestone.`
- Link duplicates or related bugs instead of creating repeated reports.

If Plans 01 through 04 found no confirmed or likely defects, verify no fake Phase 7 bug file exists and prepare a no-defect slice-row update.

#### Acceptance Criteria

- `rg -n "discovery_phase: 7" docs/bugs/BPBUG-*.md` returns all and only Phase 7 real bug reports, or returns no matches when no Phase 7 defects were found.
- Every returned Phase 7 report contains the no-fix sentence.
- Duplicate or related findings are cross-linked in the bug report body or index row.
