# Phase 7 Plan 02: Build Pipeline And Generated Assets

## Goal

Audit Blueprint's build pipeline and generated `dist` bundle for package-script coverage, declaration and bundle outputs, copied artifact schemas, tracked runtime assets, MCP startup, and generated asset freshness.

## Scope

In scope:
- `package.json` scripts and Node/TypeScript/esbuild dependency expectations.
- `scripts/build.mjs` declaration emit, bundle entrypoints, source maps, and schema copy behavior.
- Generated `dist/mcp/server.js`, built hook entrypoints, declarations, and schema files.
- Built asset smoke tests and runtime-contract tracked-file assertions.

Out of scope:
- Keeping or committing rebuilt `dist` changes as fixes.
- Modifying package scripts, source entrypoints, tests, schema files, or generated assets.
- Containerized install readiness; Plan 04 covers install smoke behavior.

## Tasks

### Task 1: Audit build script and package scripts

#### Read First

- `package.json`
- `tsconfig.json`
- `scripts/build.mjs`
- `src/mcp/server.ts`
- `src/hooks/read-before-edit.ts`
- `src/hooks/blueprint-write-guard.ts`
- `src/hooks/workflow-advisory.ts`
- `src/mcp/artifact-contracts/index.ts`

#### Action

Verify these concrete target states:

- `package.json` declares Node `>=20`, ESM `"type": "module"`, and scripts `build`, `typecheck`, `test`, `smoke:gemini-clean-home`, and `test:integration:extension`.
- `scripts/build.mjs` removes `dist/`, fails with clear `npm ci` guidance when local `typescript` or `esbuild` dependencies are missing, runs `tsc -p tsconfig.json --emitDeclarationOnly`, bundles exactly these entrypoints: `hooks/blueprint-write-guard`, `hooks/read-before-edit`, `hooks/workflow-advisory`, and `mcp/server`.
- The build target is Node 20, format is ESM, platform is Node, sourcemaps are enabled, and artifact schemas are copied from `src/mcp/artifact-contracts/schemas` to `dist/mcp/artifact-contracts/schemas`.

#### Acceptance Criteria

- `rg -n "emitDeclarationOnly|entryPoints|blueprint-write-guard|read-before-edit|workflow-advisory|mcp/server|sourcemap|node20|artifact-contracts.*schemas" scripts/build.mjs` finds the expected build contract.
- `rg -n "\"build\"|\"typecheck\"|\"test\"|\"smoke:gemini-clean-home\"|\"test:integration:extension\"|\">=20\"" package.json` finds package-script and engine evidence.
- Any build-script gap that can make host installation fail or hide generated drift is documented as a bug candidate.

### Task 2: Verify generated assets and built runtime smoke

#### Read First

- `dist/mcp/server.js`
- `dist/hooks/read-before-edit.js`
- `dist/hooks/blueprint-write-guard.js`
- `dist/hooks/workflow-advisory.js`
- `dist/mcp/artifact-contracts/schemas/phase.plan.model.schema.json`
- `tests/built-assets-smoke.test.ts`
- `tests/extension-runtime-contracts.test.ts`

#### Action

Run or document:

```bash
npm run build
npx tsx --test tests/built-assets-smoke.test.ts
```

Then inspect `git status --short`. If `npm run build` changes `dist/` or declarations, do not keep those changes as a fix. Classify the diff as:

- generated asset freshness defect if tracked built assets differ from source-generated output;
- expected build churn only if the diff is caused by environment-specific metadata and can be ruled out with evidence;
- environment/dependency blocker if build cannot run because dependencies are absent or incompatible.

Verify built asset smoke covers:

- built hook commands listed in `hooks/hooks.json` execute with exit code 0, empty stderr, `decision: "allow"`, and expected advisory messages.
- built MCP server starts over stdio and advertises the same tool names as `blueprintToolNames`.
- runtime-contract tests require key built files to be git-tracked for Git-installed extension hosts.

#### Acceptance Criteria

- `npm run build` exits 0, or the exact failure is recorded.
- `npx tsx --test tests/built-assets-smoke.test.ts` exits 0, or the exact failure is recorded.
- Any generated diff after build is recorded as evidence and not silently committed as a fix.
- Missing or untracked runtime assets, missing schema copies, or MCP/hook built smoke failures are documented as bug candidates.

#### Read First

#### Acceptance Criteria

- `git status --short` shows Plan 02 changes limited to `docs/bugs/` and `.planning/phases/07-host-packaging-build-hooks-audit/`; generated `dist/` changes are not kept as Phase 7 output.

## Verification

- Run `npm run build` or record the blocker.
- Run `npx tsx --test tests/built-assets-smoke.test.ts` or record the blocker.
- Run `find dist -maxdepth 3 -type f | sort` and compare against expected MCP, hook, declaration, map, and schema outputs.
- Run `git status --short`.

## Threat Model

| Threat | Risk | Mitigation |
|--------|------|------------|
| T-01 | Tracked `dist` is stale and installed hosts launch broken runtime code. | Rebuild and inspect generated diffs as evidence; do not preserve fixes during discovery. |
| T-02 | Build omits schemas or hook entrypoints that hosts need at runtime. | Compare `scripts/build.mjs`, `dist/`, and built smoke tests. |
| T-03 | Dependency absence creates false verification failure. | In fresh worktrees run `npm ci`; otherwise record exact missing dependency guidance from the build script. |

## Must Haves

- Build script entrypoints, declaration emit, schema copy, and package scripts are audited together.
- Generated `dist` runtime assets are checked for presence, tracked status, and executable smoke behavior.
- Any generated drift is documented as evidence only, not fixed in this milestone.
- Any bug report must include concrete command/file evidence and the no-fix sentence.
- No source, manifest, skill, test, generated asset, runtime `.blueprint/`, installed-extension, host-global, branch, PR, remote, or git-history fix is applied.
