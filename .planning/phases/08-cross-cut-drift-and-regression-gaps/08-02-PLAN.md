# Phase 8 Plan 02: Risk-Backed Regression Gap Audit

## Goal

Audit missing regression coverage across prior bugs, high-risk safety gates, generated asset freshness, and user-visible runtime contracts, then file bugs only for coverage gaps with material risk.

## Scope

In scope:
- Prior recurrence paths from BPBUG-001 through BPBUG-004.
- High-risk report-before-mutate, destructive filesystem, remote/git, generated asset, schema/report validation, and implemented-only routing guards.
- Existing nearby tests that should or should not catch each failure mode.

Out of scope:
- Adding or changing tests.
- Re-running the full suite unless targeted test evidence is inconclusive.
- Filing low-risk "more coverage would be nice" notes as bugs.
