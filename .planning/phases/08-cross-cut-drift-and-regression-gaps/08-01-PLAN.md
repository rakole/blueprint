# Phase 8 Plan 01: Cross-Layer Contract Drift Matrix

## Goal

Build the Phase 8 drift matrix and audit material cross-layer mismatches between Blueprint docs, command catalog, command manifests, skills, MCP docs, runtime/source, tests, and generated runtime-contract assets.

## Scope

In scope:
- Implemented-only routing and runtime catalog truth for `/blu`, `/blu-help`, `/blu-progress`, and `/blu-next`.
- `docs/COMMAND-CATALOG.md`, `docs/commands/*.md`, `commands/*.toml`, `skills/*/SKILL.md`, `docs/MCP-TOOLS.md`, `docs/RUNTIME-REFERENCE.md`, and live catalog/resource source.
- Cross-family drift rows for router/readiness, lifecycle, roadmap/capture/lightweight execution, review/impact/shipping, workspace/maintenance, packaging/hooks/generated assets, and shared MCP docs/tests.

Out of scope:
- Re-auditing every prior command slice from scratch when no cross-cut evidence touches it.
- Fixing source, docs, manifests, skills, tests, generated assets, or runtime behavior.
- Filing low-impact wording drift as a bug.

Create `.planning/phases/08-cross-cut-drift-and-regression-gaps/08-DRIFT-MATRIX.md` with:
