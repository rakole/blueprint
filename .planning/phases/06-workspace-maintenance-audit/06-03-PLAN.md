# Phase 6 Plan 03: Cleanup Protected-Scope Audit

## Goal

Audit `/blu-cleanup` for protected current phase and active roadmap exclusions, saved closeout evidence, report-before-mutate behavior, destination and overwrite confirmations, partial filesystem failure visibility, and discovery-only safety.

## Scope

In scope:
- `/blu-cleanup` docs, command manifest, and maintenance skill contract.
- Cleanup tool ordering across `blueprint_project_status`, `blueprint_roadmap_read`, `blueprint_artifact_list`, `blueprint_artifact_summary_digest`, `blueprint_artifact_report_write`, and `blueprint_state_update`.
- `cleanup-latest` report contract and saved report path authority.
- Cleanup metadata tests and relevant artifact/state runtime helpers.

Out of scope:
- Actually moving, deleting, archiving, or compacting any `.blueprint/` or `.planning/` phase directories.
- Fixing cleanup implementation or tests.
- Workspace create/remove and workstreams, already covered by Plans 01 and 02.

## Tasks

### Task 1: Compare cleanup prompt contracts to protected-scope rules

#### Read First

- `.planning/phases/06-workspace-maintenance-audit/06-CONTEXT.md`
- `.planning/phases/06-workspace-maintenance-audit/06-RESEARCH.md`
- `docs/commands/cleanup.md`
- `commands/blu-cleanup.toml`
- `skills/blueprint-maintenance/SKILL.md`
- `docs/MCP-TOOLS.md`
- `docs/RUNTIME-REFERENCE.md`
- `docs/ARTIFACT-SCHEMA.md`

#### Action

Verify that cleanup contracts require these concrete target states:

- `cleanup` reads `blueprint_project_status` first and stops on uninitialized, partial, or unhealthy project state.
- `cleanup` reads `blueprint_roadmap_read` before proposing any archive scope so current phase and active roadmap references stay protected.
- Dirty tree, missing phase root, or inconsistent phase layout blocks before mutation with `dirty-working-tree`, `missing-phase-root`, or `inconsistent-phase-layout`.
- Closeout evidence comes from `blueprint_artifact_list` and digest input comes from explicit repo-relative `artifactPaths` passed to `blueprint_artifact_summary_digest`.
- The preview names selected phase directories, protected exclusions, archive destination, move/copy-delete behavior, `cleanup-latest`, report overwrite posture, and exact filesystem commands before mutation.
- `cleanup-confirmation`, `archive-destination-confirmation`, and `report-overwrite-confirmation` remain visible until resolved.
- `blueprint_artifact_report_write` persists `cleanup-latest` before any filesystem mutation, and `blueprint_state_update` occurs only after report write and filesystem success.

#### Acceptance Criteria

- `rg -n "blueprint_project_status|blueprint_roadmap_read|blueprint_artifact_list|blueprint_artifact_summary_digest|blueprint_artifact_report_write|cleanup-latest|cleanup-confirmation|archive-destination-confirmation|report-overwrite-confirmation|current phase|active roadmap" docs/commands/cleanup.md commands/blu-cleanup.toml skills/blueprint-maintenance/SKILL.md docs/MCP-TOOLS.md docs/RUNTIME-REFERENCE.md docs/ARTIFACT-SCHEMA.md` finds aligned contract anchors.
- Any drift between docs, manifest, skill, runtime reference, MCP docs, and artifact schema is either ruled out with citations or recorded as a bug candidate.

### Task 2: Audit cleanup implementation substrate and test coverage

#### Read First

- `src/mcp/tools/artifacts.ts`
- `src/mcp/tools/state.ts`
- `tests/cleanup-metadata.test.ts`
- `docs/commands/cleanup.md`
- `commands/blu-cleanup.toml`
- `skills/blueprint-maintenance/SKILL.md`

#### Action

Audit whether cleanup has enough deterministic runtime and regression support for D-04 and D-06:

- `blueprint_artifact_summary_digest` must treat returned `inputsUsed` as authoritative and not invite the model to widen cleanup evidence after digest creation.
- `blueprint_artifact_report_write` must accept the bare report name `cleanup-latest`, return the authoritative path, enforce overwrite behavior, and keep report writes path-contained.
- The command/skill must require the cleanup report before filesystem mutation and preserve the report if later filesystem archiving partially fails.
- The command/skill must never archive the current phase, active roadmap references, or evidence-incomplete directories.
- The metadata tests must prove the documented contract exists. If no dedicated runtime cleanup test exists, determine whether that creates a concrete test-gap bug given the high-risk filesystem behavior.

Run or document the targeted cleanup metadata command:

```bash
npx tsx --test tests/cleanup-metadata.test.ts
```

#### Acceptance Criteria

- The targeted cleanup command exits 0, or the exact failure/environment blocker is recorded.
- Any cleanup behavior that can archive current/active phase directories, skip report-before-mutate, lose cleanup reports after partial filesystem failure, or smooth over missing closeout evidence is documented as a Phase 6 bug candidate with exact contract evidence.
- Any missing runtime coverage is documented as a test-gap bug only if source/docs inspection shows a concrete risk not covered by existing tests.

## Threat Model

| Threat | Risk | Mitigation |
|--------|------|------------|
| T-01 | The audit moves or deletes phase directories while investigating cleanup. | Do not run cleanup mutation; inspect contracts/tests only or use disposable temp roots if a probe becomes necessary. |
| T-02 | A report-before-mutate gap is missed because the command prompt names the report but the flow can mutate first. | Check exact manifest step order and skill rules, then inspect report tool semantics. |
| T-03 | Missing cleanup runtime coverage is overreported as a bug without evidence. | File a test-gap bug only if the missing coverage maps to concrete destructive risk. |

## Must Haves

- D-04, D-06, D-13, D-14, and D-15 are explicitly covered by this plan.

- Cleanup must persist `cleanup-latest` before filesystem mutation.
- Any bug report must include concrete evidence and the no-fix sentence.
- No source, manifest, skill, test, generated asset, runtime `.blueprint/`, installed-extension, host-global, workspace, branch, PR, remote, or git-history fix is applied.
