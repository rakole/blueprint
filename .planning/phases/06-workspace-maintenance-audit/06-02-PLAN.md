# Phase 6 Plan 02: Workstream Snapshot Integrity

## Goal

Audit Blueprint workstreams for project-local state ownership, canonical index/state validation, explicit waiting states, dirty-tree active transitions, resume snapshot integrity, and narrow `STATE.md` restoration.

## Scope

In scope:
- `/blu-workstreams` docs, command manifest, and maintenance skill contract.
- `blueprint_workstream_list`, `blueprint_workstream_mutate`, and final `blueprint_state_update` usage.
- `.blueprint/workstreams/WORKSTREAMS.md` and `.blueprint/workstreams/<slug>/state.json` contracts.
- Workstream metadata and tools tests.

Out of scope:
- Host-global workspace registry and workspace teardown; that is Plan 01.
- Cleanup archival behavior; that is Plan 03.
- Applying any runtime or test fixes.

## Tasks

### Task 1: Compare workstream command contracts to project-local state rules

#### Read First

- `.planning/phases/06-workspace-maintenance-audit/06-CONTEXT.md`
- `.planning/phases/06-workspace-maintenance-audit/06-RESEARCH.md`
- `docs/commands/workstreams.md`
- `commands/blu-workstreams.toml`
- `skills/blueprint-maintenance/SKILL.md`
- `docs/MCP-TOOLS.md`
- `docs/RUNTIME-REFERENCE.md`
- `docs/ARTIFACT-SCHEMA.md`

#### Action

Verify that the workstream contracts require these concrete target states:

- Workstream state stays project-local under `.blueprint/workstreams/WORKSTREAMS.md` plus `.blueprint/workstreams/<slug>/state.json`.
- No host-global workstream registry, `.planning/` ownership, or `workflow.use_workstreams` config toggle is reintroduced.
- Read-only operations use `blueprint_workstream_list` only and summarize active, paused, completed, snapshot, waiting-state, reason, and next-safe-action data without mutation.
- `create`, `switch`, `resume`, and `complete` use `blueprint_workstream_mutate` as the sole workstream write owner.
- `switch` and active-stream `complete` require explicit `ask_user` confirmation and preserve `workstream-switch-confirmation` or `workstream-archive-confirmation`.
- `resume` restores only the returned `statePatch` through `blueprint_state_update` and does not pass `lastUpdated`.
- Blocked states remain visible as `missing-workstream`, `missing-resume-snapshot`, `dirty-working-tree`, and `corrupt-workstream-index`.

#### Acceptance Criteria

- `rg -n "\\.blueprint/workstreams/WORKSTREAMS\\.md|\\.blueprint/workstreams/<slug>/state\\.json|blueprint_workstream_list|blueprint_workstream_mutate|blueprint_state_update|workstream-switch-confirmation|workstream-archive-confirmation|missing-resume-snapshot|corrupt-workstream-index|workflow\\.use_workstreams" docs/commands/workstreams.md commands/blu-workstreams.toml skills/blueprint-maintenance/SKILL.md docs/MCP-TOOLS.md docs/RUNTIME-REFERENCE.md docs/ARTIFACT-SCHEMA.md` finds aligned contract anchors.
- Any drift between docs, manifest, skill, runtime reference, MCP docs, and artifact schema is either ruled out with citations or recorded as a bug candidate.

### Task 2: Audit workstream implementation and regression coverage

#### Read First

- `src/mcp/tools/workspace.ts`
- `src/mcp/tools/state.ts`
- `tests/workstreams-metadata.test.ts`
- `tests/workstream-tools.test.ts`

#### Action

Audit whether implementation and tests satisfy D-07 through D-09:

- `blueprintWorkstreamList` must return empty, valid, blocked, project-missing, or invalid states without mutating workstream files.
- Canonical `state.json` content must validate `version`, `name`, `slug`, `status`, timestamps, and snapshot content; malformed timestamps or snapshots must block as `corrupt-workstream-index` rather than being normalized away.
- `WORKSTREAMS.md` drift from canonical per-stream state files must block as `corrupt-workstream-index`.
- `blueprintWorkstreamMutate` must keep exactly one active workstream, snapshot the current active stream before switching/resuming/completing when required, and block dirty-tree active transitions.
- `resume` must return only the saved state subset in `statePatch` and leave timestamp ownership to `blueprint_state_update`.
- Failed workstream writes must roll back newly-created directories and not leave stale index entries.

Run or document the targeted workstream test command:

```bash
npx tsx --test tests/workstreams-metadata.test.ts tests/workstream-tools.test.ts
```

#### Acceptance Criteria

- The targeted workstream command exits 0, or the exact failure/environment blocker is recorded.
- Any workstream behavior that writes outside `.blueprint/workstreams/`, smooths over corrupt canonical state, invents snapshot data, drops waiting states, or widens resume into `/blu-resume-work` is documented as a Phase 6 bug candidate with exact source/test evidence.
- Any missing regression coverage is documented as a test-gap bug only if it creates concrete risk around active stream switching, snapshot restoration, or corrupt-state handling.
