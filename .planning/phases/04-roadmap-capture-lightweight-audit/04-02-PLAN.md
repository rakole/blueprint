# Phase 4 - Plan 02

## Goal

Audit milestone report and carry-forward flows so closeout decisions remain grounded in saved evidence, report contracts, and implemented follow-up routing.

## Scope

- Compare milestone command specs against TOML manifests, roadmap-admin skill text, runtime reference, and tool documentation.
- Inspect report-writing, digest, artifact contract, scaffold, and state-readiness behavior that milestone commands rely on.
- Run focused milestone metadata/tool tests or document the environment blocker.
- Create bug reports only for confirmed or likely defects.

## Tasks

### Task 1: Audit milestone command contract alignment

#### Read First

- `docs/commands/audit-milestone.md`
- `docs/commands/complete-milestone.md`
- `docs/commands/milestone-summary.md`
- `docs/commands/new-milestone.md`
- `commands/blu-audit-milestone.toml`
- `commands/blu-complete-milestone.toml`
- `commands/blu-milestone-summary.toml`
- `commands/blu-new-milestone.toml`
- `skills/blueprint-roadmap-admin/SKILL.md`
- `docs/RUNTIME-REFERENCE.md`
- `.planning/phases/04-roadmap-capture-lightweight-audit/04-CONTEXT.md`

#### Action

- Build a checklist with these exact expected milestone requirements:
  - `/blu-audit-milestone` reads roadmap and saved artifact evidence, reads `report.milestone-audit`, writes a project-local report, and routes actionable gaps to `/blu-plan-milestone-gaps`.
  - `/blu-complete-milestone` reads `blueprint_state_load`, fails unless `derivedStatus.milestoneAudit.readyForCompletion` is true, writes `report.milestone-complete`, and routes to `/blu-milestone-summary <milestone>`.
  - `/blu-milestone-summary` requires saved audit and completion reports, writes `report.milestone-summary`, and routes to `/blu-new-milestone`.
  - `/blu-new-milestone` reads roadmap and saved milestone summary, reads `phase.context`, scaffolds carry-forward starter docs through MCP, preserves historical phase directories, and routes to `/blu-discuss-phase <first phase>`.
  - All overwrite gates and follow-up routes stay explicit and implemented-only.
- Compare docs, TOML prompts, skill text, and runtime reference for material drift.
- File a bug if drift could cause closeout from chat memory, missing readiness gates, wrong report contract, historical phase deletion, or non-implemented follow-up routing.

#### Acceptance Criteria

- `rg -n "readyForCompletion|report\\.milestone-audit|report\\.milestone-complete|report\\.milestone-summary|blueprint_artifact_contract_read|blueprint_artifact_summary_digest|blueprint_artifact_report_write|blueprint_artifact_scaffold|/blu-plan-milestone-gaps|/blu-new-milestone" docs/commands/audit-milestone.md docs/commands/complete-milestone.md docs/commands/milestone-summary.md docs/commands/new-milestone.md commands/blu-audit-milestone.toml commands/blu-complete-milestone.toml commands/blu-milestone-summary.toml commands/blu-new-milestone.toml skills/blueprint-roadmap-admin/SKILL.md docs/RUNTIME-REFERENCE.md` finds expected anchors.
- Any new bug report cites the exact mismatched contract surfaces in `## Evidence`.
