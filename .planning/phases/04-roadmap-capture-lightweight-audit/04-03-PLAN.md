# Phase 4 - Plan 03

## Goal

Audit Blueprint capture commands so notes, todos, backlog items, backlog promotion, and explore routing stay deterministic, project-local, duplicate-aware, and MCP-owned.

## Scope

- Compare capture command specs against TOML manifests, capture skill text, MCP tool docs, and runtime reference rows.
- Inspect `blueprintArtifactMutateIndex` and backlog promotion behavior.
- Run focused capture metadata/tool tests or document the environment blocker.
- Create bug reports only for confirmed or likely defects.

## Tasks

### Task 1: Audit capture command contract alignment

#### Read First

- `docs/commands/note.md`
- `docs/commands/add-todo.md`
- `docs/commands/check-todos.md`
- `docs/commands/add-backlog.md`
- `docs/commands/review-backlog.md`
- `docs/commands/explore.md`
- `commands/blu-note.toml`
- `commands/blu-add-todo.toml`
- `commands/blu-check-todos.toml`
- `commands/blu-add-backlog.toml`
- `commands/blu-review-backlog.toml`
- `commands/blu-explore.toml`
- `skills/blueprint-capture/SKILL.md`
- `.planning/phases/04-roadmap-capture-lightweight-audit/04-CONTEXT.md`

#### Action

- Build a checklist with these exact expected capture requirements:
  - `/blu-note` writes non-empty notes through `blueprint_artifact_mutate_index` with `target: "note"` and reports duplicates instead of appending copies.
  - `/blu-add-todo` writes non-empty todos through `blueprint_artifact_mutate_index` with `target: "todo"` and duplicate handling.
  - `/blu-check-todos` reads project status, lists pending todos, and mutates todo status only through exact ids or exact text matches.
  - `/blu-add-backlog` writes backlog rows through `blueprint_artifact_mutate_index`, reserves `999.x` stubs only when explicitly requested and confirmed, and scaffolds reserved context through MCP.
  - `/blu-review-backlog` previews through `blueprint_roadmap_promote_backlog`, promotes only confirmed returned backlog ids, updates backlog rows, clears consumed reservation metadata, and routes from returned promoted phase numbers.
  - `/blu-explore` classifies into exactly one target, confirms before writes, routes oversized work to `/blu-quick` or `/blu-plan-phase`, and uses MCP-owned persistence for any confirmed write.
- Compare docs, TOML prompts, skill text, MCP docs, and runtime reference for material drift.
- File a bug if drift could cause duplicate capture, wrong todo status mutation, backlog promotion from prose-derived ids, unconfirmed reserved stubs, or non-implemented follow-up routing.

#### Acceptance Criteria

- `rg -n "blueprint_artifact_mutate_index|blueprint_roadmap_promote_backlog|createdEntryIds|duplicateEntryIds|matchedEntryIds|reservedPhase|reservePhaseStub|previewOnly|/blu-quick|/blu-plan-phase" docs/commands/note.md docs/commands/add-todo.md docs/commands/check-todos.md docs/commands/add-backlog.md docs/commands/review-backlog.md docs/commands/explore.md commands/blu-note.toml commands/blu-add-todo.toml commands/blu-check-todos.toml commands/blu-add-backlog.toml commands/blu-review-backlog.toml commands/blu-explore.toml skills/blueprint-capture/SKILL.md docs/MCP-TOOLS.md docs/RUNTIME-REFERENCE.md` finds expected anchors.
- Any new bug report cites the exact mismatched contract surfaces in `## Evidence`.
