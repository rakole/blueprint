# Phase 4 - Plan 05

## Goal

Close the Phase 4 roadmap/capture/lightweight audit by checking `/blu-debug`, reconciling all Phase 4 bug reports, updating the bug index slice row, and confirming the no-fix boundary.

## Scope

- Compare debug command spec against TOML manifest, debug skill text, MCP docs, and runtime reference.
- Inspect report write, todo capture, and state routing substrates used by debug.
- Verify debug remains diagnose/report/follow-up gated and does not silently fix or capture todos.
- Reconcile Phase 4 summaries and bug docs into `docs/bugs/INDEX.md`.
- Run final `git status --short` boundary check.

## Tasks

### Task 1: Audit debug contract alignment

#### Read First

- `docs/commands/debug.md`
- `commands/blu-debug.toml`
- `skills/blueprint-debug/SKILL.md`
- `docs/MCP-TOOLS.md`
- `docs/RUNTIME-REFERENCE.md`
- `.planning/phases/04-roadmap-capture-lightweight-audit/04-CONTEXT.md`

#### Action

- Build a checklist with these exact expected debug requirements:
  - `/blu-debug` requires a concrete issue statement before deep investigation.
  - `/blu-debug --diagnose` stays diagnose-only until the user confirms a fix attempt.
  - Non-trivial investigations keep resolved scope, active stage, pending gate, execution mode, and next safe action visible.
  - Durable debug evidence persists through `blueprint_artifact_report_write` with bare report name `debug-latest`.
  - Follow-up todo capture uses `blueprint_artifact_mutate_index` only after explicit user ask or confirmation.
  - Follow-up routing stays inside implemented commands: `/blu-quick`, `/blu-plan-phase`, `/blu-validate-phase`, or `/blu-progress`.
- Compare docs, TOML prompt, skill text, MCP docs, and runtime reference for material drift.
- File a bug if drift could cause silent fixes, missing durable reports, implicit todo capture, diagnose-only boundary leakage, or non-implemented follow-up routing.

#### Acceptance Criteria

- `rg -n "concrete issue|diagnose-only|debug-latest|blueprint_artifact_report_write|blueprint_artifact_mutate_index|follow-up gate|/blu-quick|/blu-plan-phase|/blu-validate-phase|/blu-progress" docs/commands/debug.md commands/blu-debug.toml skills/blueprint-debug/SKILL.md docs/MCP-TOOLS.md docs/RUNTIME-REFERENCE.md` finds expected anchors.
- Any new bug report cites the exact mismatched contract surfaces in `## Evidence`.
