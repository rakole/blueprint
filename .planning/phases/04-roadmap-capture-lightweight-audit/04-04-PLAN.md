# Phase 4 - Plan 04

## Goal

Audit Blueprint lightweight execution so `/blu-fast` stays trivial and report-free while `/blu-quick` stays bounded, report-backed, and honest about next safe routing.

## Scope

- Compare fast and quick command specs against TOML manifests, execution skill text, runtime references, MCP docs, and runtime reference rows.
- Inspect report write and state routing substrates relevant to quick-run evidence.
- Run focused lightweight execution tests or document the environment blocker.
- Create bug reports only for confirmed or likely defects.

## Tasks

### Task 1: Audit fast and quick contract alignment

#### Read First

- `docs/commands/fast.md`
- `docs/commands/quick.md`
- `commands/blu-fast.toml`
- `commands/blu-quick.toml`
- `skills/blueprint-phase-execution/SKILL.md`
- `skills/blueprint-phase-execution/references/fast-runtime-contract.md`
- `skills/blueprint-phase-execution/references/quick-runtime-contract.md`
- `skills/blueprint-phase-execution/references/long-running-execution-profile.md`
- `docs/RUNTIME-REFERENCE.md`
- `.planning/phases/04-roadmap-capture-lightweight-audit/04-CONTEXT.md`

#### Action

- Build a checklist with these exact expected lightweight execution requirements:
  - `/blu-fast` starts from project status, qualifies only trivial work, uses no subagents, no tracker/todo layer, no durable quick-run report, no phase artifacts, and optional state refresh only when the repo is initialized.
  - `/blu-fast` reroutes oversized execution asks to `/blu-quick` or `/blu-plan-phase`.
  - `/blu-quick` remains bounded, uses optional depth gates only when requested or needed, persists `quick-run-latest` through `blueprint_artifact_report_write`, and updates state through `blueprint_state_update`.
  - `/blu-quick` does not create hidden saved plans, summaries, phase completion claims, or lifecycle execution claims.
  - Both commands keep follow-up guidance inside implemented commands.
- Compare docs, TOML prompts, skill text, runtime references, MCP docs, and runtime reference rows for material drift.
- File a bug if drift could cause broad unplanned mutation, missing quick-run evidence, false lifecycle completion, report overwrite risk, or non-implemented follow-up routing.

#### Acceptance Criteria

- `rg -n "no subagents|no quick-run report|quick-run-latest|blueprint_artifact_report_write|does not create hidden saved plan|does not impersonate|bounded quick|/blu-plan-phase|/blu-progress" docs/commands/fast.md docs/commands/quick.md commands/blu-fast.toml commands/blu-quick.toml skills/blueprint-phase-execution/SKILL.md skills/blueprint-phase-execution/references/*.md docs/MCP-TOOLS.md docs/RUNTIME-REFERENCE.md` finds expected anchors.
- Any new bug report cites the exact mismatched contract surfaces in `## Evidence`.

### Task 2: Audit lightweight execution report/state substrate and tests

#### Read First

- `src/mcp/tools/artifacts.ts`
- `src/mcp/tools/state.ts`
- `src/mcp/artifact-contracts/index.ts`

- `tests/quick-metadata.test.ts`
- `tests/lightweight-execution-regression.test.ts`
- `tests/lifecycle-pilot-integration.test.ts`

#### Action

- Inspect report write support for `quick-run-latest` and any report contract requirements that prevent hand-built `.blueprint/reports/` paths.
- Inspect state update/routing expectations relevant to fast and quick completion.
- Verify tests distinguish `/blu-fast` from `/blu-quick`: fast must not persist `quick-run-latest`, quick must persist the bare report name, and neither should impersonate saved phase execution.
- Run targeted tests:
  - `npx tsx --test tests/fast-metadata.test.ts tests/quick-metadata.test.ts tests/lightweight-execution-regression.test.ts`
- If tests fail, classify whether the failure is defect evidence, an environment blocker, or unrelated pre-existing breakage.

#### Acceptance Criteria

- `rg -n "quick-run-latest|report\\.quick-run|blueprintArtifactReportWrite|blueprint_state_update|doesNotMatch\\(.*quick-run-latest|does not create" src/mcp/tools/artifacts.ts src/mcp/tools/state.ts src/mcp/artifact-contracts/index.ts tests/fast-metadata.test.ts tests/quick-metadata.test.ts tests/lightweight-execution-regression.test.ts tests/lifecycle-pilot-integration.test.ts` finds handler and test evidence.
