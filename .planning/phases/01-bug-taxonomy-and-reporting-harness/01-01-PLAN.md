# Phase 1 - Plan 01

## Goal

Create the canonical bug-report authoring contract at `docs/bugs/TEMPLATE.md` so every later audit phase records Blueprint defects with stable ids, required evidence, consistent classification, and explicit no-fix discipline.

## Scope

- Create `docs/bugs/` if it does not exist.
- Add `docs/bugs/TEMPLATE.md`.
- Define the required report filename pattern `BPBUG-###-short-slug.md`.
- Define required frontmatter keys: `id`, `title`, `severity`, `confidence`, `surface`, `status`, `discovery_phase`, and `reported`.
- Define required sections: `Summary`, `Classification`, `Expected Behavior`, `Actual Behavior`, `Impact`, `Affected Files`, `Evidence`, `Verification Steps`, `Likely Cause`, `Suggested Fix Direction`, `Uncertainty`, `Related Bugs`, and `No Fix Applied`.
- Define exact vocabularies for severity, confidence, affected surface, and status.
- Include explicit Blueprint boundary language: Blueprint is a Gemini-native extension, `.blueprint/` is runtime state, and `.planning/` is local audit bookkeeping.

#### Action

- Create directory `docs/bugs/` if missing.
- Create `docs/bugs/TEMPLATE.md`.
- Add an H1 title `# Blueprint Bug Report Template`.
- Add a `Filename and ID` section stating that real bug files must use `BPBUG-###-short-slug.md`, real ids start at `BPBUG-001`, and `BPBUG-000` is reserved for the illustrative non-real example.
- Add a YAML frontmatter example with the exact keys `id`, `title`, `severity`, `confidence`, `surface`, `status`, `discovery_phase`, and `reported`.
- Add a `Blueprint Boundary` section with these exact facts: `Blueprint is a Gemini-native CLI extension`, `Blueprint runtime state is .blueprint/`, and `.planning/ is local audit bookkeeping`.

### Task 3: Add evidence and no-fix authoring rules

#### Read First

- `docs/bugs/TEMPLATE.md`
- `.planning/PROJECT.md`
- `.planning/REQUIREMENTS.md`
- `.planning/phases/01-bug-taxonomy-and-reporting-harness/01-VALIDATION.md`

#### Action

- In `## Evidence`, require a consolidated table with columns `Source`, `Evidence`, and `Why It Matters`.
- In `## Verification Steps`, require numbered reproduction or verification steps and the expected observable outcome.
- In `## Uncertainty`, require a statement even for `confidence: confirmed`, using `None known` only when direct evidence exists.
- In `## No Fix Applied`, include this exact sentence: `No source, manifest, skill, test, generated asset, or runtime behavior fix was applied during this discovery milestone.`
- Add a probe cleanup note requiring temporary probe files to be removed and documented when any are used.

#### Acceptance Criteria

- `rg -n "Source.*Evidence.*Why It Matters|Verification Steps|expected observable outcome|None known|temporary probe files" docs/bugs/TEMPLATE.md` finds matches.
- `rg -n "No source, manifest, skill, test, generated asset, or runtime behavior fix was applied during this discovery milestone." docs/bugs/TEMPLATE.md` finds the exact no-fix sentence.
