# Phase 1 - Plan 02

## Goal

Create `docs/bugs/INDEX.md` as a rich triage board that later audit phases can update consistently with bug rows, duplicate links, lifecycle status, discovery phase, impact, likely cause, and report links.

## Scope

- Add `docs/bugs/INDEX.md`.
- Define real bug numbering rules and reserve `BPBUG-000` for the illustrative example.
- Add the main bug table with columns required by D-05 and HARN-04.
- Add concise vocabulary references for severity, confidence, affected surface, and lifecycle status.
- Add no-fix and git-status hygiene guidance for later phases.
- Add a section for duplicate and related-bug handling as forward-compatible guidance for Phase 9, without claiming later-phase requirement coverage.

## Tasks

### Task 1: Create the index shell and bug table

#### Read First

- `.planning/REQUIREMENTS.md`
- `.planning/ROADMAP.md`
- `.planning/phases/01-bug-taxonomy-and-reporting-harness/01-CONTEXT.md`
- `.planning/phases/01-bug-taxonomy-and-reporting-harness/01-PATTERNS.md`
- `docs/COMMAND-CATALOG.md`

#### Action

- Create `docs/bugs/INDEX.md`.
- Add H1 `# Blueprint Bug Index`.
- Add a `Purpose` section stating that the index is a triage board for the Blueprint Defect Discovery Milestone.
- Add a `Bug Reports` table with exactly these columns: `ID`, `Title`, `Severity`, `Confidence`, `Surface`, `Status`, `Discovery Phase`, `Impact`, `Likely Cause`, and `Report`.
- Add a placeholder row for the next real bug id `BPBUG-001` with status guidance such as `next real bug id`, not a fake finding.
- State that `BPBUG-000` is reserved for the illustrative non-real example and is excluded from real defect totals.

#### Acceptance Criteria

- `test -f docs/bugs/INDEX.md` exits 0.
- `rg -n "^# Blueprint Bug Index|Blueprint Defect Discovery Milestone" docs/bugs/INDEX.md` finds matches.
- `rg -n "\\| ID \\| Title \\| Severity \\| Confidence \\| Surface \\| Status \\| Discovery Phase \\| Impact \\| Likely Cause \\| Report \\|" docs/bugs/INDEX.md` finds the table header.
- `rg -n "BPBUG-001|BPBUG-000|excluded from real defect totals" docs/bugs/INDEX.md` finds matches.

### Task 2: Add vocabularies, dedupe, and no-fix guidance

#### Read First

- `docs/bugs/INDEX.md`
- `.planning/PROJECT.md`
- `.planning/REQUIREMENTS.md`
- `.planning/phases/01-bug-taxonomy-and-reporting-harness/01-RESEARCH.md`

#### Action

- Add `## Vocabularies` with exact lists for severity, confidence, affected surface, and status.
- Add `## Duplicate And Related Findings` explaining that duplicates should be marked with `status: duplicate` and linked to the canonical bug instead of silently repeated.
- Add `## Discovery-Only Guardrails` stating this exact sentence: `No source, manifest, skill, test, generated asset, or runtime behavior fixes are applied during this milestone.`
- Add `## Phase Boundary Hygiene` requiring later phases to run `git status --short` and confirm changes are limited to `.planning/` and `docs/bugs/` before completion.

#### Acceptance Criteria

- `rg -n "critical.*high.*medium.*low.*info|confirmed.*likely.*suspected|command.*skill.*MCP tool.*hook.*docs.*tests.*build.*generated assets.*state artifacts.*host behavior.*cross-cutting" docs/bugs/INDEX.md` finds matches.
- `rg -n "new.*triaged.*planned.*in-progress.*fixed.*verified.*closed.*duplicate.*closed-invalid" docs/bugs/INDEX.md` finds the status vocabulary.
- `rg -n "status: duplicate|canonical bug|No source, manifest, skill, test, generated asset, or runtime behavior fixes are applied during this milestone|git status --short" docs/bugs/INDEX.md` finds matches.

### Task 3: Add slice tracking and handoff rules

#### Read First

- `docs/bugs/INDEX.md`
- `.planning/ROADMAP.md`
- `.planning/REQUIREMENTS.md`

#### Action

- Add `## Slice Coverage` with a checklist or table for phases 1 through 9.
- For each phase row, include columns or fields for `Surfaces Examined`, `Surfaces Deferred`, and `Bug IDs`.
- State that later phases should update their slice row when they complete even if no bugs are found.
- State that planned-only or non-routable Blueprint commands must not be recommended as immediate remediation paths.

#### Acceptance Criteria

- `rg -n "Slice Coverage|Surfaces Examined|Surfaces Deferred|Bug IDs" docs/bugs/INDEX.md` finds matches.
- `rg -n "Phase 2|Phase 3|Phase 4|Phase 5|Phase 6|Phase 7|Phase 8|Phase 9" docs/bugs/INDEX.md` finds later phase rows.
- `rg -n "planned-only|non-routable Blueprint commands" docs/bugs/INDEX.md` finds the routing guardrail.
