# Phase 1 - Plan 03

## Goal

Seed one realistic but clearly non-real bug report at `docs/bugs/BPBUG-000-illustrative-example.md`, update the index to reference it as illustrative only, and run final harness checks that prove Phase 1 stayed discovery-only.

## Scope

- Create `docs/bugs/BPBUG-000-illustrative-example.md` using the template from Plan 01.
- Mark the example as `ILLUSTRATIVE ONLY`, `not a real Blueprint defect`, `status: closed-invalid`, and excluded from real defect totals.
- Use a plausible Blueprint scenario such as a fictional command/help mismatch, but avoid naming it as confirmed evidence.
- Update `docs/bugs/INDEX.md` with a clearly separated illustrative-example note or row.
- Run final file-read, grep, and git-status checks for the Phase 1 harness.

## Tasks

### Task 1: Write the illustrative non-real report

#### Read First

- `docs/bugs/TEMPLATE.md`
- `docs/bugs/INDEX.md`
- `.planning/phases/01-bug-taxonomy-and-reporting-harness/01-CONTEXT.md`
- `.planning/phases/01-bug-taxonomy-and-reporting-harness/01-RESEARCH.md`

#### Action

- Create `docs/bugs/BPBUG-000-illustrative-example.md`.
- Use frontmatter with `id: BPBUG-000`, `severity: info`, `confidence: suspected`, `surface: docs`, `status: closed-invalid`, `discovery_phase: 1`, and `reported: 2026-05-01`.
- Put `ILLUSTRATIVE ONLY - not a real Blueprint defect` directly under the H1.
- Fill every required template section with plausible but clearly fictional evidence.
- In `## No Fix Applied`, include the exact sentence from the template: `No source, manifest, skill, test, generated asset, or runtime behavior fix was applied during this discovery milestone.`

#### Acceptance Criteria

### Task 3: Run final harness verification and boundary check

#### Read First

- `docs/bugs/TEMPLATE.md`
- `docs/bugs/INDEX.md`
- `docs/bugs/BPBUG-000-illustrative-example.md`
- `.planning/phases/01-bug-taxonomy-and-reporting-harness/01-VALIDATION.md`

- Run `rg -n "critical.*high.*medium.*low.*info|confirmed.*likely.*suspected|new.*triaged.*planned.*in-progress.*fixed.*verified.*closed.*duplicate.*closed-invalid" docs/bugs`.
- Run `git status --short`.
- Confirm any changed paths are limited to `.planning/` and `docs/bugs/` for this phase. If unrelated user changes already exist, do not revert them; report them separately.

#### Acceptance Criteria

- The `test -f` command exits 0 for all three docs.
- The no-fix `rg` command finds matches in `docs/bugs/TEMPLATE.md`, `docs/bugs/INDEX.md`, and `docs/bugs/BPBUG-000-illustrative-example.md`.
- The vocabulary `rg` command finds the required vocabularies in the harness docs.
- `git status --short` output is inspected and no Phase 1 execution changes touch source, manifests, skills, tests, build outputs, generated assets, runtime `.blueprint/`, or host-global state.
