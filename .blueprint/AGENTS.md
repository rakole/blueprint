# Blueprint Planning Instructions

Use this area for Blueprint planning artifacts, not for product specs unrelated to the extension itself.

## Planning Rules

- Keep plans in `.blueprint/plans/`.
- Prefer small execution slices that map to real scaffold gaps.
- Capture assumptions, dependencies, and verification criteria explicitly.
- Write plans for extension work such as commands, hooks, agents, skills, MCP integration, and policies.
- Avoid ceremonial planning documents that do not guide the next implementation move.
