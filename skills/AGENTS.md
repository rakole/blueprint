# Blueprint Skill Instructions

Skills are lightweight instruction bundles that shape how Blueprint handles work.

## Rules

- Keep one skill per directory with a `SKILL.md` file.
- Preserve valid YAML frontmatter at the top of each skill file.
- Describe when the skill should be used before describing how it behaves.
- Keep workflow steps short, concrete, and execution-oriented.
- Align skill language with the Blueprint workflow terms used elsewhere in the repo.
- Do not promise automation or orchestration that the extension does not implement yet.
