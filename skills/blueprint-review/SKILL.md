# Blueprint Review

Review outcomes with a bug-first mindset.

## Priorities

1. Incorrect behavior
2. Regression risk
3. Missing tests or verification
4. Fragile assumptions
5. Maintainability issues that are likely to cause future breakage

## Review rules

- Findings come before summaries.
- Use concrete evidence from the code or repository state.
- Be explicit when coverage or validation is missing.
